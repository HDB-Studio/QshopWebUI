## QshopWebUI

**跨服务器 QS 商店网页查询与可视化平台**

(https://github.com/HDB-Studio/QshopWebUI)

### 📖 项目简介

QshopWebUI 是一个**开源、跨服务器**的 QS (QuickShop) 商店网页查询系统，专为 Minecraft 服务器玩家设计。通过标准化 API 接口，任何搭载 QS 插件的服务器均可接入，让玩家无需安装 Mod 即可在网页端高效查找、比价和定位商店。

> 🏗️ **建筑党的福音** — 告别游戏内盲目寻找商店，网页端一键比价，省时省力。

---

### ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🔌 **跨服务器兼容** | 通过配置 API 端点，支持任意 QS 插件服务器的接入 |
| 🧠 **智能价格过滤** | 基于加权历史均价算法，自动排除天价物品与个人宝库 |
| 🎮 **3D 物品渲染** | 网页端交互式 3D 物品展示，支持 NBT 数据兼容 |
| 🔍 **多维排序** | 支持按价格 / 数量 / 距离（港口）排序显示 |
| 🔐 **安全传输** | TLS 1.3 + 混合加密架构，保障 API 数据安全 |
| 📦 **本地缓存** | 可选本地数据库缓存，提升查询速度，支持离线浏览 |

---

### 🧮 智能过滤算法

我们设计了一套基于历史交易数据的价格合理性评估系统：

```
加权历史均价 = Σ(单次历史成交价 × 单次采购量) / Σ(所有历史采购量)
价格合理度比值 = 当前定价 / 加权历史均价
```

**判定规则**：比值在 **0.6 ~ 1.5** 区间内视为合理价格(可設定)，自动过滤异常定价，确保搜索结果的实用性。

### 🚀 快速开始

#### 服务器端（安装过滤 Mod）
1. 将过滤 Mod 放入服务器的 mods 文件夹
2. 配置过滤参数与 API 输出设置
3. 启动服务器，Mod 将自动暴露过滤后的 QS 数据接口

#### 网页端部署
1. 克隆本仓库
2. 配置你的服务器 API 端点信息
3. 按项目文档完成部署（支持 Docker、静态托管、Node 服务器等多种方式）

#### 接入新服务器
在配置文件中添加服务器 API 信息即可接入，无需修改核心代码。

### 🛠️ 技术栈

- **前端**: React / Vue3 + Three.js (3D 渲染)
- **后端**: Node.js / Python FastAPI (可选本地缓存服务)
- **数据**: QS 插件 API → 过滤 Mod → RESTful API
- **安全**: TLS 1.3 + AES-GCM 混合加密
